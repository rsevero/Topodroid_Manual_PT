Métodos de nomeamento de bases

Quando você recebe dados via bluetooth o TopoDroidX pode automaticamente atribuir nomes de bases às medidas recebidas. Uma medição repetida é considerada uma visada, as outras são tratadas como medidas auxiliares. Se você seguir um método ao fazer medições, o nomeamento automático de bases do TopoDroidX pode ser muito útil. O TopoDroidX suporta uma série de convenções para nomear automaticamente as bases das medidas (tanto auxiliares como visadas). Você pode selecionar aquela que se encaixa em seu método de trabalho através da opção de configuração Dados topo > Nomear bases.

Os métodos disponíveis para nomear automaticamente bases são:

  1. nenhum: desabilita a funcionalidade de nomeamento automático de bases do TopoDroidX.
  2. aux, visada: na base DE você obtém inicialmente as medidas auxiliares e depois a visada (medidas repetidas para a nova base a ser criada). Este é o método padrão.
  3. aux, contra: na base PARA você obtém primeiro as medidas auxiliares e depois a contra visada (medidas repetidas para a base anterior).
  4. contra, aux, visada: na base PARA você inicialmente obtém a contra visada (medidas repetidas para a base anterior). Em seguida obtém as medidas auxiliares a partir da base PARA inicial e, por último, a visada para a próxima base (medidas repetidas para a nova base a ser criada).
  5. tripé: é similar ao método anterior — contra, aux, visada — mas visa diminuir a quantidade de locais onde o tripé precisa ser instalado reduzindo a quantidade de locais onde medidas serão produzidas. Por exemplo: na base 1 você obtém medidas auxiliares e faz a visada para a base 2. Move o tripé para a base 3, pulando a base 2 e faz a contra visada para 2, as medidas auxiliares em 3 e a visada para 4. Move o tripé diretamente para a base 5, pulnado a base 4 e faz a contra visada para 4, as medidas auxiliares em 5 e a visada para 6 e assim sucessivamente. [A]
  6. anomalia magnética: as medidas devem ser obtidas da mesma forma que no método 3 acima — contra, aux, visada — mas o tratamento dos dados obtidos é realizado de uma forma diferente, detalhada abaixo. [E]
  7. visada, aux: na base DE você obtém primeiro a visada (medidas repetidas para a nova base a ser criada) e depois as medidas auxiliares.
  8. contra, aux: na base PARA você obtém primeiro a contra visada (medidas repetidas para a base anterior) e depois as medidas auxiliares.
  9. TopoRobot: funciona como o método 1 acima — aux, visada — mas os nomes de bases criados seguem a convenção do TopoRobot. [T]

Exemplos de como devem ser realizadas as medidas para cada um dos métodos descritos acima (as medidas entre parênteses são obtidas na mesma base):

  1. aux, visada: ... (1-. 1-. 1-. 1-2) (2-. 2-. 2-3) ...
  2. aux, contra: ... (2-. 2-. 2-. 2-1) (3-. 3-. 3-2) ...
  3. contra, aux, visada: ... (2-1 2-. 2-. 2-3) (3-2 3-. 3-. 3-4) ...
  4. tripé: ... (2-1 2-. 2-. 2-3) (4-3 4-. 4-. 4-5) ...
  5. anomalia magnética: ... (2-1 2-. 2-. 2-3) (3-2 3-. 3-. 3-4) ... (igual ao método "3")
  6. visada, aux: ... (1-2 1-. 1-. 1-.) (2-3 2-. 2-.) ...
  7. contra, aux: ... (2-1 2-. 2-. 2-.) (3-2 3-. 3-.) ...
  8. TopoRobot: ... (1-. 1-. 1-. 1-2) (2-. 2-. 2-3) ... (igual ao método "1")

Base de origem

Cada vez que o nomeamento automático de bases é ativado, ele usa como base de origem a última base criada (ou penúltima para os métodos contra e tripé). Se a topo ainda não tem nenhuma base criada, usa como base de origem a base inicial da topo (definida no momento da criação da topo). Portanto, as medidas já existentes não são modificadas, e todas as novas medidas obtidas serão atribuídas a bases poteriores a última base criada. Isso pode afetar a base de medidas auxiliares imediatamente após a última visada, quando o método de nomeamento automático des bases é alterado.

A base de origem pode ser escolhida manualmente, tanto na tela Medidas topo (através de um clique longo sobre o nome da base escolhida) quanto na tela Desenho (clicando no botão Aplicar na caixa de diálogo Base). A base de origem selecionada manualmente é apresentada em verde em ambas as telas.

Método contra, aux, visada

At each station you take the backshot to the previous station, next the splay shots, and finally the forward shot to the next station.
Backsight shots are checked: if a shot is not close to the opposite of the previous leg, it is not considered a backsight shot.
The backsight shots are marked "duplicate".

Tripod policy
At station "1" you take splays and then shots to the station "2". Then you move into station "3", take shots to station "2", then splays, then shots to station "4". Move to station "5" and so on.

Magnetic anomaly compensation [Tester level]
The magnetic anomaly compensation is done estimating at each station the "local" magnetic declination by comparing the azimuths of the leg reaching to the station and the backsight leg. The azimuth of all the shots at the station are then corrected with the "local" declination. An overall declination remains because the declination at the first station cannot be determined. Furthermore the survey azimuth angles become relative as in a traverse, therefore the angle errors add up instead of compensating.
The magnetic anomaly compensation requires taking backsight shots. However, with magnetic anomaly compensation the check whether a shot is a backsight does not take the azimuth into account. The magnetic anomaly compensation is carried out during the data reduction (for the sketches).

TopoRobot policy
The "TopoRobot" policy is just like the first, but station names follow the TopoRobot convention.

Station naming policy and activity levels
Certain station naming policy are only enabled at high activity levels:

  • "Tripod" requires "advanced" level
  • "Magnetic anomaly" requires "expert" level
  • "TopoRobot" requires "tester" level

Reducing the activity level when one of these is active, automatically set the station policy to the default policy (splays + foreshot). The policy is automatically reinstated again if the activity level is raised to the required value or above.

Blunder leg-shot [T]
A blunder shot is an intermediate leg shot that happened to be off-target. Typical leg sequences with blunders 'B' are "A-B-A-A" and "A-A-B-A".
Blunder leg-shot detection is an experimental feature of the default station naming policy. It must be enabled with a custom setting.
Blunder shots are detected and marked as blank entry that will be hidden when the list is refreshed.
Blunder shots can be recovered.
