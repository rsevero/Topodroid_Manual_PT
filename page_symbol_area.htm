Ferramentas de desenho de área

As ferramentas de desenho de área são definidas em arquivos de texto.

Duas ferramentas de área, "usuário" e "água", são definidas internamente no aplicativo. As outras são carregadas da pasta privada area do TopoDroidX. Este diretório é populado com as ferramentas de área empacotadas no arquivo apk de instalação do aplicativo. Você pode excluir as ferramentas que não precisa, editar as que deseja alterar e adicionar novas que não são fornecidas no apk.

Os arquivos de definição de ferramentas de área contém linhas especificando as propriedades da ferramenta de desenho:

  • symbol area: linha de abertura que identifica um arquivo de definição de ferramenta do tipo área [obrigatória].
  • name NAME onde NAME é o nome padrão da área [obrigatório].
  • nomes traduzidos: por exemplo, "name-pt", "name-es", etc. [opcional].
  • th_name NAME: onde NAME é o nome Therion. O nome do arquivo deve ser o mesmo que o nome Therion, sem o possível prefixo "u:" [obrigatório].
  • color RGB ALPHA: RGB e ALPHA, em formato hexadecimal, definem a cor e a opacidade usadas ao desenhar a área em questão. Por exemplo: '0x0000ff' é azul, e '0x99' é semi-transparente. [obrigatório].
  • close-horizontal: falso se ausente [opcional].
  • orientable: falso se ausente [opcional].
  • level: indica a quais camadas de visualização esta ferramenta está associada. 1 se ausente [opcional].
  • bitmap WIDTH HEIGHT X_TILE Y_TILE: início do padrão de bitmap usado para representar a área na tela de desenho do croqui. Os modos de repetição podem ser M (espelho) ou R (repetir). Esta linha deve ser seguida por HEIGHT linhas de comprimento WIDTH com os pixels do bitmap (1: cor definida em color acima, 0: cor de fundo).
  • endbitmap: marca o final de um bitmap.
  • roundtrip VALUE [opcional, para SVG-roundtrip VALUE pode ser 1 (walls_shp), 2 (walls_sym), 3 (detail_shp), 4 (detail_sym) - padrão 3].
  • endsymbol: finaliza o arquivo de definição de área.

Nome

Consulte a página Ferramenta de ponto para saber como especificar os nomes.

Close-horizontal

Se este atributo estiver definido, a borda da área é fechada horizontalmente na vista de perfil e nos cortes.

Se a coordenada Y do último ponto da linha desenhada estiver próxima à do primeiro ponto, a borda da área é interrompida no primeiro ponto que tem a coordenada Y igual à do primeiro ponto. Dessa forma, você pode desenhar áreas com uma linha superior (ou inferior) perfeitamente horizontal. Se as coordenadas Y dos pontos inicial e final da área desenhada diferirem muito, a borda é fechada com um segmento inclinado.

Atualmente, este atributo é definido apenas para a ferramenta interna de desenho de área tipo "água".

Bitmap pattern

Com esta opção, você pode especificar um padrão de fundo, por exemplo, para diferenciar formações rochosas.

Os padrões de área não são usados no conjunto de símbolos padrão. A coloração chapada definida através do atributo color é preferido por questões de desempenho à opção de padronagem definida através do atributo bitmap.

Este atributo não é usado em nenhuma ferramenta disponível no arquivo apk de instalação do aplicativo.

Orientação

Se o atributo "orientable" estiver definido, o padrão de área pode ser rotacionado através da caixa de diálogo de edição de área.

Level

O atributo "level" define o valor da flag que especifica em quais camadas os itens desta área devem ser apresentados.

A flag é um número que é a soma de 1 (Base), 2 (Fundo), 4 (Decoração), 8 (Cobertura) e 16 (Avulsos). Veja mais detalhes em Camadas.
