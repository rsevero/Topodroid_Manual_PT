Geração de logs

Arquivos de log são importantes nos momentos em que o app não se comporta conforme esperado. Por exemplo quando o resultado de um cálculo não é igual ao obtido por outros meios ou quando os dados informados pelo usuário não são devidmente reconhecidos.

A geração de arquivos de log é controlada através da opção de menu Log na tela Principal.

Os arquivos de log podem ser enviados para o log do sistema ou para um arquivo de log, que é salvo na pasta base do TopoDroidX. O log do sistema pode ser inspecionado a partir de um PC via USB com programas como "adb". O arquivo de log pode ser aberto em modo "anexar", com mensagens de log anexadas aos logs existentes, ou em modo de "sobrescrever".

Existem várias opções de log para limitar os logs àqueles necessários para encontrar a causa do problema. Por padrão, apenas mensagens do tipo "erro" (ou seja, quando o programa detecta algo que não deveria ter acontecido) são registrados.

Os logs [E] são diferentes dos travamentos do software. Os travamentos são devidos ao programa se comportando mal no sistema e são detectados pelo sistema, que encerra o programa. Você deve relatar travamentos (e ANR) através de um email para o desenvolvedor para que um rastreamento de pilha seja anexado e eles possam ser corrigidos mais facilmente.

Consulte o site do TopoDroidX para saber como relatar travamentos e ANR.
