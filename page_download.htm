DOWNLOAD DADOS

O botão Download inicia uma conexão com o dispositivo para baixar dados. Enquanto a conexão estiver ativa, o botão permanece vermelho. Se você estiver usando o tipo de conexão sob demanda a conexão será fechada quando não houver mais dados a serem baixados. No tipo de conexão contínua a conexão permanece aberta (e dados são continuamente baixados assim que são produzidos) até que a conexão entre o dispositivo e o aparelho Android seja perdida.

Qual modo é mais adequado depende dos seus hábitos e do tipo de dispositivo usado.

Com o modo sob demanda você pode se concentrar em desenhar e outras tarefas sem ser incomodado por novos dados sendo capturados e, quando decidir baixar novos dados, poderá se concentrar neles (estarão destacados em azul). Por outro lado o modo sob demandaexige pausas para obter novos dados e essas pausas podem ser longas se várias medidas foram capturas desde o último download.

Quando a conexão Bluetooth é perdida (por exemplo se o dispositivo ficar longe demais do Android ou for desligado):

  • no modo sob demanda: o download de dados é interrompido.
  • no modo contínua: o TopoDroidX tenta reestabelecer a conexão repetidamente em intervalos de poucos segundos.

Quando o tela Dados topo é fecahda a conexão para download de dados é encerrada. A conexão de download não é interrompida quando a tela é pausada (por exemplo quando a tela do Android é desligada). Se você abrir um croqui enquanto a conexão de download estiver ativa no modo contínua a conexão permanece ativa e dados seguem sendo baixados (e apresentados no croqui).

Para o SAP5 e o BRIC4 a conexão é sempre do tipo contínua.

DistoXBLE

O download de dados da DistoXBLE pode falhar na primeira tentativa imediatamente após o Bluetooth do Android tiver sido habilitado. Nesse caso desligue e religue a DistoXBLE e tente novamente fazer o download de dados.

BRIC4

O download de dados da BRIC4 pode ser limitado somente aos valores essenciais (distância, azimute, inclinação e pouco mais) ou ser completo com todas as informações transferidas pelo protocolo da BRIC4.

No modo completo é incluido o índice da medida. Por padrão o TopoDroidX usa o índice do BRIC4 como índice de dados no banco de dados, se recebê-lo do BRIC4. Como o índice de dados precisa ser único, se o TopoDroidX receber uma medida com um índice já existente no banco de dados, um novo valor será usado no lugar do valor recebido.

O índice BRIC4 pode ser reiniciado para seu valor original através da opção de menu Advanced menu >> Memory clear do BRIC4.

É possível desabilitar o uso do índice do BRIC4 [T] fazendo com que o TopoDroidX atribua valores incrementais aos dados baixados.
