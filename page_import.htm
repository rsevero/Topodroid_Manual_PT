Importar topo

O TopoDroidX importa dados de topo de arquivos zip criados pelo TopoDroidX ou de arquivos em alguns outros formatos.

A caixa de diálogo de improtação é apresentada ao clicar-se no botão Importar na tela Principal. O arquivo a ser importado é selecionado através da tela padrão de seleção de arquivos do Android. O nome do arquivo a ser importado não pode conter o caracter "dois pontos" ':'.

A caixa de diálogo de importação apresenta a lista de formatos suportados:

  • Zip: arquivos zip criados pelo próprio TopoDroidX
  • Compass: dat
  • Survex: svx
  • Therion: somente medidas, th
  • VisualTopo: tro e trox
  • Walls: srv
  • TopoRobot
  • PocketTopo: incluindo croquis, top
  • CaveSniper: csn
  • CSV da BRIC: csv

A importação de alguns formatos pode ser configurada através de opções específicas.

A importação falha se já houver uma topo com o mesmo nome no TopoDroidX.

Na importação de arquivos em formatos que contém dados EDTC (esquerda/direita/teto/chão), o TopoDroidX converte-os para medidas auxiliares (apresentadas em verde).

Compass (dat)

  • Importar EDTC: quando habilitada, os valores das medidas EDTC são importados e convertidos para medidas auxiliares.
  • Visada antes de EDTC: quando habilitada, as medidas EDTC são importadas após a respectiva visada.
  • Modo de dados 'mergulho': habilita importação de arquivos no modo 'mergulho'. [T]

O TopoDroidX converte os arquivos do compass do modo normal para o modo mergulho [T] se a opção Especiais >> Importar/Exportar >> Form dado importação estiver configurada como mergulho [G].

Survex (svx)

A sintaxe dos arquivos Survex é parcialmente suportada.

Therion (th)

Somente arquivo Therion com dados de medidas em formato de dados "normal" são suportados. O comando "declination" é considerado: os valores de aximute lidos são corrigidos pela declinação informada. Se os nomes das bases não se chocarem com nomes já existentes, as bases podem ser importadas somente com seus nomes originais. Do contrário, deverão ser importadas com nomes extendidos, ou seja, com o nome da topo adicionada ao nome original da base. O TopoDroidX não importa croquis do Therion.

VisualTopo (tro, trox)

  • Formato de arquivo Trox
  • Importar EDTC: quando habilitada, os valores das medidas EDTC são importados e convertidos para medidas auxiliares.
  • Visada antes de EDTC: quando habilitada, as medidas EDTC são importadas após a respectiva visada.

O TopoDroidX pode também importar topos criadas como Prof no lugar de Clino no formato "trox". Os dados importados são armazenados em formato mergulho.

PocketTopo (top)

O TopDroidX importa tanto os dados de medidas como os croquis do PocketTopo. O processo de importação faz uma conversão das cores usadas no PocketTopo para os tipos de pontos e linhas do TopoDroidX. Essa conversão pode ser configurada através da opção Importar/Exportar >> Importar >> Mapa cor PocketTopot.

CSV da BRIC (csv)

Dados importados a partir de arquivos CSV criados pela BRIC contém somente distância, azimute e inclinação. Os dados da topo importados precisam ser editados para que sejam incluídas as outras informações desejadas.

Observação

A importação pode falhar (e derrubar o APP) devido a um erro no banco de dados que o TopoDroidX não consiga interceptar.
