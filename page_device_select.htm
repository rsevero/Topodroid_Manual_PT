SELECIONAR DISPOSITIVO

O TopoDroidX suporta os seguintes dispositivos:

  • DistoX: primeira versão
  • DistoX2: segunda versão
  • DistoXBLE: terceira versão
  • SAP5: (Shetland Attack Pony), somente os modelos com Bluetooth
  • BRIC4: (Bluetooth Ruggerized Integrated Cartographer)

Todos os dispositivos devem ser previamente pareados com o Android.

Normalmente o TopoDroidX trabalha com um só dispositivo por vez. Os dados podem ser baixados continuamente (modo contínuo) ou em lotes (modo por demanda). A diferença entre os dois modos é que no modo contínuo permanece ativo após o download dos dados ter terminado ou mesmo a conexão ao dispositivo ter sido fechada enquanto no modo por demanda o download é finalizado.

SAP5 e BRIC4 somente funcionam no modo contínuo.

Com as DistoXs o TopoDroidX pode trabalhar com mais de um dispositov ao mesmo tempo apesar de que os dados sõ são baixados de um dispositivo por vez.

O modo de download de dados deve ser alterado para multi. Quando isso é feito, um clique longo no botão download abre a caixa de diálogo para escolher de qual DistoX se quer baixar dados.

SAP5 - Shetland Attack Pony

O TopoDroidX tem suporte para o Shetland Attack Pony, versão 5 (SAP5) com bluetooth.

O dispositivo deve ter sido previamente pareado com o Android usando o app Configurações do Android. O código PIN é "000000" (seis zeros).

A partir desse momento o SAP5 será apresentado na lista de dispositivos conhecidos na tela Dispostivos. Aparecerá listado como "SAP XX MAC_ADDRESS" onde "XX" é o código de duas letras do SAP5, por exemplo "HG" e MAC_ADDRESS é o endereço MAC do SAP5.

Para selecionar o SAP5 como dispostivo atual. clique nela na lista de dispositivos conhecidos.

Enquanto conectado, o SAP5 vai transferir as medidas realizadas para o app, independente dos dados serem ou não armazenados no dispositivo.

O protocolo de transmissão do SAP5 não inclui uma confirmação do recebimento dos dados, dessa maneira:

  • se o SAP5 não estiver conectado, os dados não são transferidos para o app.
  • no momento em que o SAP5 for conectado os dados ainda não transferidos não serão transferidos para o app.

O SAP5 mantém os dados capturados em sua memória interna e podem ser obtidos através de um arquivo Survex, usando o software "SAP PonyTrainer".

A calibração do SAP5 é realizada através do próprio dispostivo e não utiliza um app externo.

BRIC4

O TopoDroidX tem suporte para o BRIC versão 4.

O dispositivo deve ter sido previamente pareado com o Android usando o app Configurações do Android.

A aprtir desse momento o BRIC4 será apresentado na lista de dispositivos conhecidos na tela Dispostivos. Aparecerá listado como "Bric4 CODE MAC_ADDRESS" onde CODE é o código de 4 dígitos do BRIC4, por exemplo "0039" e MAC_ADDRESS é o endereço MAC do BRIC4.

Para selecionar o BRIC4 como dispositivo atual é só clicar na linha dele na lista de dispositivos conhecidos.

A partir do momento em que tiver sido conectado a um aparelho Android, o BRIC4 lembra da conexão, mesmo após ter sido desligado permanecendo, dessa maneira, "sempre conectado".

Quando o TopoDroidX é conectado, as medidas armazenadas no BRIC4 são imediatamente transferidas para o Android. Se o TopoDroidX não estiver conectado, os dados são mantidos em memória e transferidos quando o app for aberto.

O protocolo de transmissão do BRIC4 incluí a confirmação do recebimento dos dados pelo Android mas não incluí a confirmação de recebimento dos dados pelo app. O protocolo tem um atraso de alguns segundos entre a conexão e a primeira transmissão de dados e o TopoDroidX tem um atraso de um segundo entre o recebimento do último dado antes de fechar a conexão. Dessa maneira normalmente nenhum dado será perdido mas isso não é garantido e medidas são perdidas se o app não for avisado pelo Android antes de que a próxima medida seja recebida.

O BRIC4 armazena os dados na sua memória interna em arquivos csv. Se conectado a um computador através de um cabo USB, o dispositivo aparece como um dispositivo de memória externo e os arquivos csv podem ser copaidos para o computador.

A calibração do BRIC4 é realizada pelo prórpio dispositivo e não depende de um app externo.

O BRIC4 pode ser controlado através do app. é possível ligar e desligar o laser além de disparar a captura de medidas. O BRIC4 tem também o modo "escanear" onde medidas são capturadas continuamente, uma por segundo aproximadamente.

O TopoDroidX pode usar três protocolos de dados diferentes (escolhidos através da opção de configuração Especiais >> Dispositivos >> Modo BRIC):

  • Só principais: recebe somente os dados principais (distância, azimute e inclinação) não incluindo o ID da medida atribuído pelo BRIC4.
  • Todos com índice: recebe todos os dados e usa o ID atribuído pelo BRIC4.
  • Todos sem índice: recebe todos os dados mas não usa o ID atribuído pelo BRIC4. O TopoDroidX cria seu próprio índice.

Quando todos os dados são recebidos, o TopoDroidX armazena os códigos de erro do BRIC4 e seus valores no comentário da medida.
