BANCOS DE DADOS TopoDroidX

Pode ser útil ter uma ideia geral sobre a organização das informações e dos dados nos bancos de dados usados pelo TopoDroidX. As unidades de trabalho do TopoDroidX são as topos e as calibrações. Uma topo representa uma única sessão de topo de um pedaço da caverna. A calibração é uma só calibração DistoX.

As topos são armazenadas no banco de dados "distox14.sqlite". Há um banco de dados como esse em cada diretório de projeto.

As calibrações são armazenadas no banco de dados "device10.sqlite" junto com as informações do dispositivo calibrado. Esse banco de dados é único e fica armazenado no diretório privado do TopoDroidX "files".

A tabela mais importante no banco de dados de topos é surveys que armazena os dados da topo.

Os registros das topos armazenam o nome, data, equipe, declinação magnética base inicial e eventualmente um comentário sobre a topo.

Outras tabelas relacionadas às topos:

  • shots: dados da topo
  • stations: bases gravadas
  • plots: croquis 2D: plantas baixas, perfis e cortes
  • photos; imagens assocaidas a visadas
  • audios; áudiocomentários relacionados a visadas
  • sensors: medidas de sensores diversos associadas a visadas
  • fixeds: bases georeferênciadas através de coordenadas

As tabelas mais importantes do banco de dados de calibração são:

  • devices: informações sobre dispositivos
  • calibs: informações sobre calibrações

Os registros de calibrações contém o nome, data, nome da DistoX, algoritmo usado e resultados da última calibração: coeficientes, erro médio, erro máximo e quantidade de iterações.

Relacionada às calibrações há a tabela gms: dados de calibração.

Por último cada banco de dados tem uma tabela configs: valores de configuração.

Você pode aprender mais sobre a organização dos dados nos bancos de dados analisando-os com um visualizador de dados SQLite.
